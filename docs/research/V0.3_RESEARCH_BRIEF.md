# V0.3 Research Brief: Research Gates

## Question

What should Project Governor add after v0.2 `upgrade-advisor`?

## Conclusion

V0.3 should add two advisory-only research gates:

1. `version-researcher`: analyzes candidate releases before upgrade-advisor makes a recommendation.
2. `research-radar`: evaluates candidate capabilities, agent patterns, and ecosystem changes before implementation.

This is the right next step because v0.2 can advise on upgrade choices, but it cannot yet explain evidence quality, skipped releases, whether changes match current project needs, or whether a proposed capability should be adopted now, isolated as a spike, watched, or rejected.

## Evidence summary

- Codex plugins are the installable distribution unit for skills and related integrations.
- Codex skills are the workflow unit for reusable instructions, references, and optional scripts.
- Codex subagents can parallelize research and review, but must be explicitly requested.
- Codex automations can use skills and can run in worktrees for safer scheduled maintenance.
- Claude-style memory patterns support the distinction between persistent project rules and agent-authored notes; Project Governor should keep mandatory rules checked into the repo.

## V0.3 scope

Adopt now:

- Add `version-researcher` skill.
- Add `research-radar` skill.
- Add deterministic scripts for release research and candidate scoring.
- Add research and release-research templates.
- Add prompts for both workflows.
- Extend self-tests.

Defer:

- Strict blocking hooks mode, because hooks can create false positives and should remain optional.
- Fully automatic skill generation, because candidates should remain reviewable before becoming skills.
- Local memory as source of truth, because team rules should live in `AGENTS.md` and checked-in docs.

## Acceptance criteria

- The plugin exposes both research skills.
- Version research reports skipped versions, evidence quality, relevant changes, risks, and user choices.
- Research radar ranks candidates as `adopt_now`, `spike`, `watch`, or `reject`.
- Initialization templates include research policy and release research policy files.
- Self-tests pass.

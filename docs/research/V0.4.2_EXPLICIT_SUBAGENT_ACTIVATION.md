# V0.4.2 Research Brief: Explicit Subagent Activation

## Decision

Project Governor should initialize project-scoped subagent roles and model routing so the user does not need to manually specify subagents for every workflow.

## Problem

The v0.4.1 workflow described subagents, but users still had to understand when and how to trigger them. That creates learning cost and inconsistent behavior.

## Change

V0.4.2 adds:

- `subagent-activation` skill
- project-scoped `.codex/agents/*.toml` templates
- `.codex/config.toml` subagent limits
- model-routing policy
- deterministic subagent selector
- tests for subagent mode and model choice

## Rule

Simple tasks remain fast: `micro_patch` uses no subagents by default.

Complex tasks become automatic: standard/risky/refactor/migration/upgrade/PR/research workflows run `subagent-activation`, spawn selected read-only agents, wait, then proceed.

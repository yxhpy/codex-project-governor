# V0.5.0 Research Brief: GPT-5.5 Auto Orchestration

## Decision

Project Governor should add GPT-5.5-aware automatic orchestration.

## Why

GPT-5.5 improves complex coding, tool use, research, and multi-step task completion, but Project Governor should not simply make every task run heavier. The correct optimization is model routing, context routing, and workflow routing.

## Accepted scope

- `gpt55-auto-orchestrator`
- `context-indexer`
- `clean-reinstall-manager/scripts/apply_latest_runtime_mode.py`
- project-owned runtime mode file
- context index and session brief generation
- tests covering model routing, minimal context retrieval, and clean reinstall all/project modes

## Rejected for this release

- copying `.codex/agents` or plugin-global assets into every project
- forcing GPT-5.5 high reasoning for micro patches
- reading all initialization docs in every session
- background feature implementation without user request

## Direction

This version shifts Project Governor from "many explicit skills" to "automatic skill orchestration" while keeping route guards, quality gates, and project hygiene intact.

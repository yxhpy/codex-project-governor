# awesome-design-md usage policy

Use the awesome-design-md catalog as an aesthetic reference collection, not as a license to clone another product's brand.

Allowed:

- Borrowing structure: spacing density, typography hierarchy, surface layering, CTA prominence, dashboard density, documentation readability.
- Choosing a broad archetype: developer minimalism, fintech trust, dark technical console, warm editorial workspace, consumer photography-led marketplace.
- Creating a project-specific DESIGN.md inspired by one or two references.

Not allowed unless the project owns the brand or the user explicitly asks:

- Copying logos, names, protected product copy, proprietary assets, or exact page compositions.
- Mixing many references into a visually incoherent page.
- Using a third-party brand identity in production without review.

When style is underspecified, choose at most:

- one primary aesthetic reference
- one secondary reference for a narrow concern, such as docs readability or data density

Then convert the choice into project tokens and prose in DESIGN.md.

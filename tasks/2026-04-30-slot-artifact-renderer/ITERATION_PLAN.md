# Iteration Plan

## User request

Implement the slot-based governance artifact workflow so fixed template content is rendered by deterministic scripts, while agents provide only variable slots and small update patches. Start with ITERATION_PLAN and preserve support for updating a plan during task execution.

## Existing behavior

- Agents currently create or update tasks/<date>-<slug>/ITERATION_PLAN.md directly from a Markdown template.
- Fixed headings, tables, default copy, and empty rows are repeatedly generated by the model.
- context-indexer indexes task Markdown files as ordinary task history, so generated template text can be treated as meaningful context.
- Deterministic scripts already live in tools/ and skills/*/scripts/, use the Python standard library, and expose stable JSON output contracts.

## Existing patterns to reuse

| Pattern | Source file | How it will be reused |
| --- | --- | --- |
| Deterministic CLI helper | tools/*.py, skills/*/scripts/*.py | Add stdlib-only render/update scripts with JSON input and JSON output. |
| Template payload source | templates/tasks/_template/ITERATION_PLAN.md | Keep human-readable Markdown shape compatible with the existing task artifact. |
| Context index generation | skills/context-indexer/scripts/build_context_index.py | Detect generated Markdown and index source slots instead of fixed template text. |
| Self-test style | tests/test_harness_v6.py, tests/selftest.py | Add focused unittest coverage for rendering, patching, and indexing. |

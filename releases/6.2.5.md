# 6.2.5 - Fast Path v2

## Added

- Added `tiny_patch` routing for low-risk local fixes that are larger than a `micro_patch` but still do not need full context packs, file-backed task plans, test-first artifacts, subagents, or evidence manifests.
- Added `artifact_policy` to `task-router` and `gpt55-auto-orchestrator` outputs so workflows can distinguish no-artifact, inline-evidence, and file-backed governance modes.
- Added a `tiny_patch` doc pack and targeted tests for dashboard-copy, local-widget-fix, and real widget-feature routing boundaries.

## Changed

- `micro_patch` now uses direct edit, route guard, and quality gate without creating task/test/pattern/evidence files.
- `tiny_patch`, `docs_only`, and `test_only` use inline final-response evidence instead of mandatory file-backed task artifacts.
- `standard_feature` remains the route that requires test-first planning by default; risky and UI changes keep stronger gates.
- Updated AGENTS and quality policy templates so initialized projects can adopt the faster path without weakening non-trivial workflow requirements.

## Compatibility

- Existing CLI entry points, skill names, template paths, and deterministic helpers remain compatible.
- `artifact_policy` is an additive JSON output field.
- Existing projects can receive the Fast Path v2 policy wording through plugin upgrade planning.
- No dependencies, package manifests, lockfiles, or runtime services were added.

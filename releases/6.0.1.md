# 6.0.1 — DESIGN.md Aesthetic Governor

## Added

- Added `design-md-aesthetic-governor` for UI/frontend review, prototyping, and implementation gates.
- Added `design_env_check.py` for full-service Gemini/Stitch configuration checks.
- Added remote Stitch MCP configuration at `https://stitch.googleapis.com/mcp` using `STITCH_MCP_API_KEY` without storing secrets in `.mcp.json`.
- Added `design_service_smoke.py` real-service checks for OpenAI-compatible Gemini endpoints and remote Stitch MCP initialization.
- Added `GEMINI_PROTOCOL=auto|openai|gemini` so projects can choose OpenAI-compatible gateways or native Gemini `generateContent`.
- Added `design_service_review.py` to record repeatable full-service Gemini/Stitch review evidence before UI implementation.
- Added guidance and diagnostics for native Gemini gateway roots such as `/gemini/v1beta`.
- Added `DESIGN_BASIC_MODE=1` so users can intentionally run frontend work in basic mode without Gemini/Stitch setup.
- Added project-local Codex hooks for UI prompt reminders and UI edit preflight checks.
- Added DESIGN.md aesthetic policy docs, prompt snippets, examples, release metadata, and tests.

## Workflow

- Full-service mode: GPT/Codex orchestrates, Stitch MCP supports visual prototype exploration, Gemini reviews design quality against `DESIGN.md`, then GPT/Codex implements and verifies.
- Basic mode: GPT/Codex skips Stitch/Gemini and still uses `DESIGN.md`, bundled lint, token discipline, and drift checks.
- Remote Stitch MCP needs no local npm or gcloud install when the hosted endpoint is used.

## Compatibility

- No new third-party Python dependencies.
- `.env-design` is local-only secret configuration and must not be committed.
- Existing governed projects can opt into the hook and policy templates through the plugin upgrade migrator or clean refresh.

# 6.2.2 - Diagnostics Baseline Cleanup

## Added

- Added `skills/context-indexer/scripts/build_context_roles.py` so context role classification and session brief rendering live outside the main index builder.
- Added `skills/task-router/scripts/task_router_config.py` and `skills/task-router/scripts/task_router_policy.py` so route constants, doc packs, budgets, workflows, and subagent policy can be maintained separately from classification logic.
- Added `skills/design-md-aesthetic-governor/scripts/design_service_http.py` for shared DESIGN.md service HTTP, SSL, MCP initialize, and response preview behavior.
- Added `tests/selftest_helpers.py` to keep self-test fixtures and repeated assertions below engineering standards thresholds.

## Changed

- Refactored touched deterministic helpers, hooks, and tests to clear the engineering standards baseline without changing public JSON output schemas.
- Updated context indexing so `docs/architecture/**` entries receive the `architecture` role and can satisfy route-specific doc packs.
- Updated DESIGN.md service smoke client identity to derive its version from `.codex-plugin/plugin.json` instead of hard-coded stale version strings.
- Updated diagnostics task evidence to record full test, compile, engineering standards, harness doctor, quality-gate, and context-index refresh results.

## Compatibility

- Existing skill names, template paths, deterministic helper CLI entry points, and JSON output schemas remain compatible.
- No initialized-project migration is required for this release.
- Codex and Claude Code plugin metadata, README install snippets, and Claude marketplace examples now point at `v6.2.2`.

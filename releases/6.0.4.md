# 6.0.4 - Local Marketplace Git Update Path

## Added

- Added `tools/install_or_update_user_plugin.py` for user-level Project Governor install/update workflows.
- Added local marketplace entry creation/update so users do not need to hand-edit `~/.agents/plugins/marketplace.json`.
- Added dirty checkout protection: the updater stops before fetching or checking out a new ref when local plugin edits are present.
- Added regression tests for dry-run planning, Git clone/update behavior, marketplace entry output, and dirty checkout blocking.

## Workflow

- New users can run the installer from the GitHub raw URL, then restart Codex and install Project Governor from the personal marketplace.
- Existing local marketplace users can run the raw-url installer once:
  `python3 /tmp/install_or_update_user_plugin.py --ref v6.0.4 --apply`
- After v6.0.4 is present, users can run the updater from the installed plugin checkout:
  `python3 ~/.codex/plugins/codex-project-governor/tools/install_or_update_user_plugin.py --ref v6.0.4 --apply`
- After updating the plugin checkout, use `plugin-upgrade-migrator` inside initialized projects when governance files need migration.

## Compatibility

- No new third-party Python dependencies.
- Marketplace entries remain `source: local`; built-in Git marketplace upgrade commands still do not fetch local checkouts.
- Existing initialized project migration JSON remains unchanged.

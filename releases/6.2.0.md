# 6.2.0 - Claude Code Compatibility Adapter

## Added

- Added Claude Code plugin metadata at `.claude-plugin/plugin.json`.
- Added Claude Code `pg-*` command wrappers for initialization, route/context retrieval, quality checks, memory search, upgrades, DESIGN.md preflight, and diagnostics.
- Added Claude Code subagent definitions under `claude/agents/`, aligned with existing `.codex/agents/*.toml` names and descriptions.
- Added Claude Code plugin hooks for UI/frontend prompt reminders, UI edit preflight checks, and final governance hints.
- Added Claude Code marketplace metadata and example marketplace JSON.
- Added `templates/CLAUDE.md`, which imports `AGENTS.md` so initialized projects share one governance rule source across Codex and Claude Code.
- Added `docs/compat/CLAUDE_CODE.md` with component mapping, install notes, and maintenance rules.

## Changed

- `tools/init_project.py` now treats `CLAUDE.md` as a clean-profile project-owned governance template.
- `templates/AGENTS.md` now uses cross-agent wording where Claude Code imports the same project rules.
- Public README and architecture docs now describe Codex and Claude Code plugin surfaces.

## Compatibility

- Existing Codex plugin behavior remains unchanged.
- Existing deterministic helper scripts remain dependency-free.
- Claude adapter files reference plugin-root scripts with `${CLAUDE_PLUGIN_ROOT}` so installed cache paths work.
- Existing initialized projects can add `CLAUDE.md` through `plugin-upgrade-migrator`; user-modified governance files remain untouched.

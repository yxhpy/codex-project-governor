# 6.0.2 — Governed Memory Search

## Added

- Added `context-indexer --memory-search --auto-build` for lightweight governed memory/history lookup without raw chat transcript scanning or complex shell pipelines.
- Added `--format text` to `query_context_index.py` for concise human-readable search results.
- Added `task_history` and `governance_history` roles so task plans, project state, release notes, upgrade docs, and research docs can be retrieved as project history.

## Workflow

- Use default context search for implementation context.
- Use `--memory-search` when the user asks why a decision was made, what happened before, what repeated mistake exists, or which task/release introduced behavior.
- Search curated governance artifacts such as `docs/memory/`, `docs/decisions/`, `tasks/`, `.project-governor/state/`, release notes, upgrade docs, and research docs instead of raw conversation transcripts.

## Compatibility

- No new third-party Python dependencies.
- Existing context-index JSON output is additive; default queries keep prior behavior.
- Existing governed projects can use the feature by refreshing the plugin and running `query_context_index.py --memory-search --auto-build`.

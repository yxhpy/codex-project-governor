# Project Governor Harness v6.0.6

## Added

- Generated `.project-governor/context/DOCS_MANIFEST.json` during context index writes.
- Added Markdown section indexing with summaries, line ranges, token estimates, and active/draft/stale/superseded status.
- Added `recommended_sections`, `must_read_sections`, `progressive_read_plan`, `context_compression`, and `avoid_docs` to context queries.
- Added route-specific doc packs and context budget gates to `task-router` and `gpt55-auto-orchestrator`.
- Added context-pack output for section line ranges, token budgets, compression policy, and stale/superseded docs to avoid.

## Changed

- Agents now prefer `DOCS_MANIFEST.json`, session brief, memory search, and section ranges before full documents.
- Docs marked `stale` or `superseded` are excluded by default unless `--include-stale` is supplied.
- Updated `AGENTS.md` templates so existing projects can see the new context-navigation policy through upgrade planning.

## Fixed

- The DESIGN.md UI gate and PreToolUse hook now honor `DESIGN_BASIC_MODE=1` from project-root `.env-design`, which avoids Windows or Codex hook process-inheritance failures when a shell variable is set after startup.

## Compatibility

- No new third-party dependencies.
- Existing JSON output fields remain additive.
- Existing projects can rebuild generated context with `python3 skills/context-indexer/scripts/build_context_index.py --project . --write`.

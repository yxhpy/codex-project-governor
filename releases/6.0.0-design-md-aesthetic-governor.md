# 6.0.0 addendum — DESIGN.md Aesthetic Governor

## Added

- `design-md-aesthetic-governor` skill for UI/frontend work that must read `DESIGN.md` before editing.
- Compact `awesome-design-md` aesthetic catalog for inspiration selection without copying protected brand identity.
- `design_md_gate.py` preflight that writes `.codex/design-md-governor/read-proof.json`.
- `design_env_check.py` preflight support that requires Gemini and Stitch configuration through environment variables or project-root `.env-design`.
- `verify_design_usage.py` drift check for raw hex colors and Tailwind palette classes in changed UI files.
- Repo/template Codex hook script that blocks UI edits when `DESIGN.md` read proof is missing, stale, or failed.
- AGENTS snippet and Codex prompt template for target-project adoption.

## Relationship to existing `design-md-governor`

- `design-md-governor` remains the opt-in DESIGN.md lint/summarize/diff skill.
- `design-md-aesthetic-governor` is the stricter UI coding gate: read proof first, aesthetic choice second, implementation last.

## Notes

- Hooks require `codex_hooks = true` in an active Codex config layer.
- UI review, prototyping, and UI coding are blocked until `GEMINI_BASE_URL`, `GEMINI_API_KEY`, `GEMINI_MODEL`, and `STITCH_MCP_API_KEY` are available from shell environment variables or `.env-design`.
- A user can intentionally use basic mode for a session with `DESIGN_BASIC_MODE=1`; legacy `DESIGN_ENV_SKIP=1` and `DESIGN_SERVICE_CONFIG_SKIP=1` are accepted.
- Project-local hooks load only for trusted repositories.
- `PreToolUse` is a guardrail, not a complete security boundary; final verification still runs through the skill scripts.

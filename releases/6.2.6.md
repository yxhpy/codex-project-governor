# 6.2.6 - Automatic Agent Entrypoints

## Added

- Added automatic Project Governor context injection for natural Claude Code project-work prompts, including English and Chinese compatibility, review, validation, docs, testing, upgrade, and UI wording.
- Added test coverage that verifies initialized `CLAUDE.md` files and the Claude skill wrapper do not require users to explicitly invoke `/pg-*` commands for normal governed work.
- Added `CLAUDE.md` rule-template drift detection to plugin-upgrade-migrator so already initialized projects can see the new automatic-entrypoint wording without overwriting local rules.

## Changed

- Updated the target-project `CLAUDE.md` template so Claude Code treats `/pg-*` commands as optional shortcuts and diagnostics instead of required entrypoints.
- Updated Claude hook guidance to prefer local project files and Project Governor deterministic scripts before unrelated web, browser, or MCP tools.
- Updated documentation to describe Project Governor as automatic for normal Codex and Claude project work while keeping slash commands available for inspection and diagnostics.

## Compatibility

- Codex skill names, Claude command names, plugin paths, template paths, and deterministic helper JSON schemas remain compatible.
- No dependencies, package manifests, lockfiles, or runtime services were added.
- Existing initialized projects can surface the new `CLAUDE.md` automatic-entrypoint wording through plugin-upgrade-migrator rule-template drift planning.

# 6.2.1 - Skill Entrypoint Consolidation

## Added

- Added `skills/CATALOG.json` as the machine-readable skill catalog for user-facing, internal, diagnostic, and deprecated visibility metadata.
- Added `tools/analyze_skill_catalog.py` and supporting analyzer modules to audit catalog health, README grouping, resolved entrypoint consolidations, and advisory consolidation candidates.
- Added analyzer tests that guard catalog completeness, README alignment, resolved consolidation groups, and analyzer file-size budgets.

## Changed

- Reduced the recommended skill surface to a smaller set of grouped entry points while keeping all existing skills available.
- Consolidated the Codex plugin default prompts from skill-by-skill choices into compact scenario-level prompts.
- Improved task routing so docs-only typo/edit requests route through a lighter path, while feature requests with tests do not get misclassified as test-only work.
- Split the skill catalog analyzer into smaller production modules so engineering standards pass without source-size warnings.

## Compatibility

- Existing skill names, deterministic helper schemas, template paths, and project migration metadata remain compatible.
- No initialized-project migration is required for this release.
- Claude Code adapter metadata and marketplace examples now point at `v6.2.1`.

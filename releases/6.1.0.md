# 6.1.0 - Engineering Standards Governance

## Added

- `engineering-standards-governor` skill.
- `skills/engineering-standards-governor/scripts/check_engineering_standards.py`, a dependency-free scanner for:
  - source file size
  - function length
  - approximate function complexity
  - production imports from mocks, fixtures, test data, or test-only libraries
  - mock-like production identifiers
  - test files that define cases without assertion markers
- `docs/quality/ENGINEERING_STANDARDS_POLICY.md` and copied project template.
- `tasks/_template/ENGINEERING_STANDARDS_REPORT.md`.
- Project prompt shortcut for engineering standards checks.

## Changed

- Coding routes include `engineering-standards-governor` before `quality-gate`.
- `quality-gate` accepts an optional `engineering_standards` result.
- Strict quality gates block unresolved engineering-standards warnings.
- Test planning now requires normal, boundary, error, regression, integration/contract, frontend interaction, and explicit not-tested rationale rows when relevant.
- Pattern reuse detection now includes mocks, fixtures, and test data as `test_double` reuse candidates.

Unmodified installed files may be updated from templates. User-modified governance files stay manual review or three-way merge.

## Compatibility

- No new third-party dependencies.
- Existing quality-gate inputs remain valid; `engineering_standards` is optional.
- The scanner exits non-zero only when blocking findings exist.

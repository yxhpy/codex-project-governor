# 6.0.3 — AGENTS Rule Template Drift Migration

## Added

- Added automatic `AGENTS.md` rule-template drift detection to `plugin-upgrade-migrator`.
- Added `review_rule_template_drift` operations when the installed `templates/AGENTS.md` hash differs from the latest plugin template.
- Added tests proving unchanged installed `AGENTS.md` files can be safely replaced from the latest template while user-modified files remain manual review or three-way merge items.

## Workflow

- Use `plugin-upgrade-migrator` as usual after refreshing the plugin.
- If a governed project has an install manifest with an older `AGENTS.md` template hash, the migration plan now surfaces the rule update even when no explicit release migration names `AGENTS.md`.
- Apply safe operations for unchanged installed files, then review or three-way merge locally edited `AGENTS.md` files.

## Compatibility

- No new third-party Python dependencies.
- No template paths changed.
- Existing migration plan JSON remains additive; consumers should tolerate the new `review_rule_template_drift` operation.

# 6.2.3 - Slot-Based Governance Artifacts

## Added

- Added `tools/render_governance_artifact.py` to render generated governance Markdown from structured variable slots.
- Added `tools/update_governance_artifact.py` to apply small revision-checked update patches to slot files and optionally re-render Markdown.
- Added `templates/artifacts/ARTIFACT_TEMPLATES.json` and `templates/artifacts/schemas/iteration_plan_v1.schema.json` as the machine-readable contract for the first generated artifact template.
- Added focused tests for artifact rendering, plan patch updates, and context-index generated Markdown slot indexing.

## Changed

- Updated `iteration-planner`, root and template `AGENTS.md`, and iteration contracts so generated task artifacts prefer `*.slots.json` plus deterministic render/update scripts.
- Updated `context-indexer` so generated Markdown with a `generated_from` marker is indexed from its source slot JSON when that source exists beside the Markdown file.
- Updated README guidance and API contracts for slot-based plan creation and mid-task plan updates.
- Recorded durable memory and risk-register entries for generated artifact behavior.

## Compatibility

- Existing direct `ITERATION_PLAN.md` files remain readable.
- Generated Markdown falls back to normal Markdown indexing when the source slot file is missing.
- No dependencies, package manifests, lockfiles, or runtime services were added.
- Initialized projects should review updated `AGENTS.md` and iteration-contract rules through the normal plugin upgrade migrator; user-modified rules should not be blindly overwritten.

# 6.2.4 - Actual-Project Reliability Gates

## Added

- Added host-runtime authorization awareness to subagent planning so automatic selection is separate from permission to spawn agents.
- Added the standard subagent consent sentence to the default Harness prompt.
- Added `tools/new_governance_artifact.py` so initial governance artifacts can be created from structured slots and then rendered to Markdown.
- Added `skills/quality-gate/scripts/check_execution_policy.py` and a copied runtime `EXECUTION_POLICY.json` template for command-transport constraints.
- Added a default `release_publish` execution policy that requires `gh release` or `gh api` and blocks plain `git push` unless an explicit override is approved.

## Changed

- `quality-gate` can now run execution-policy checks when `execution_context` or an execution-policy result is present.
- `gpt55-auto-orchestrator` now surfaces execution-policy context for release publishing requests that specify `gh` or GitHub API transport.
- `clean-reinstall-manager` now applies all runtime JSON templates from `templates/.project-governor/runtime/` instead of hard-coding only `GPT55_RUNTIME_MODE.json`.
- `plugin-upgrade-migrator` surfaces missing required runtime policy files for already initialized projects.
- Updated AGENTS, quality policy, README, memory, and API contracts so user-selected tools such as `gh` are treated as command policy instead of soft memory hints.

## Compatibility

- Existing CLI entry points, skill names, and initialized-project behavior remain compatible.
- The new quality-gate fields are additive.
- Existing projects can receive `.project-governor/runtime/EXECUTION_POLICY.json` through plugin upgrade planning or clean runtime refresh.
- No dependencies, package manifests, lockfiles, or runtime services were added.
